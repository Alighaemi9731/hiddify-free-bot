# 🎁 ربات کانفیگ رایگان روزانه‌ی هیدیفای

رباتی برای توزیع **کانفیگ رایگان روزانه** از پنل (یا چند پنل) **Hiddify**، با موتور
**جوین اجباری / ممبرگیری سهمیه‌دار**، سیستم **دعوت (رفرال)**، **بکاپ خودکار** و یک
**پنل مدیریت کامل داخل تلگرام**.

نوشته‌شده با **Go** (تک‌باینری، بدون وابستگی)، دیتابیس **SQLite**، تأییدشده روی
**Hiddify Manager v12**.

---

## ✨ امکانات

**برای کاربر**
- 🎁 هر روز یک کانفیگ رایگان (ریست با ساعت ایران، نیمه‌شب به وقت تهران)
- 📦 سقف حجم روزانه (پیش‌فرض ۵۰۰ مگابایت) که با **هر دعوت موفق** زیاد می‌شود (تا سقف ۳ گیگ)
- 🔗 لینک دعوت اختصاصی + شمارش دعوت‌ها
- 📊 مشاهده کانفیگ فعلی و وضعیت حساب
- 📣 درخواست تبلیغ کانال / ارتباط با پشتیبانی

**برای ادمین (منوی مدیریتی جدا)**
- 📊 داشبورد: تعداد کاربر، کانفیگ امروز، مصرف هر پنل، برترین دعوت‌کننده‌ها
- 📣 مدیریت کانال‌های تبلیغ (صف ممبرگیری): افزودن، حذف، **سهمیه‌ی جوین**، **اولویت**،
  فعال/غیرفعال، و آمار «هدف / انجام‌شده / باقی‌مانده»
- 💰 **سیستم سفارش و درآمد**: برای هر کانال **قیمت هر ۱۰۰۰ جوین** و نام سفارش‌دهنده ثبت کن؛
  ربات **درآمد** هر سفارش و کل را حساب می‌کند و موقع **تکمیل سفارش به ادمین اعلان** می‌دهد
- 🔐 **کانال خصوصی (join-request)**: ربات درخواست عضویت کانال‌های خصوصی را خودکار **تأیید** می‌کند
- 🚨 **هشدار سلامت پنل + گزارش روزانه**: اعلان خاموشی/کمبود ظرفیت پنل + خلاصه‌ی آمار و درآمد هر روز
- 🎯 شمارش فقط **جوین‌های جدید** (اگر کاربر از قبل عضو بوده، به سهمیه‌ی سفارش حساب نمی‌شود)
- 🎲 نمایش رندومِ N کانال از بین کانال‌ها برای هر کاربر در هر روز (قابل تنظیم)
- 🖥 چند پنل: افزودن پنل فقط با **لینک ادمین + یک نمونه لینک ساب**، سقف حجم روزانه برای هر پنل
- ⚙️ تنظیمات: حجم پایه، پاداش دعوت، سقف، تعداد کانال روزانه، نوع لینک ساب، **بستن ثبت‌نام**،
  **حالت تعمیر**، آیدی پشتیبانی
- 👥 مدیریت کاربر: مسدود/رفع‌مسدود، پاداش دستی، سقف ثابت
- 📢 پیام همگانی (هر نوع پیام: متن/عکس/فایل)
- 💾 بکاپ خودکار هر ۲ ساعت در تلگرام + **ریستور** با ارسال فایل بکاپ (بازگردانی کامل روی نصب خام)
- 👮 افزودن/حذف ادمین

## 🚀 نصب (اوبونتو) — تک‌خطی

```bash
curl -fsSL https://raw.githubusercontent.com/Alighaemi9731/hiddify-free-bot/main/install.sh | bash
```

نصب‌کننده **توکن ربات** و **آیدی عددی ادمین** را در ترمینال می‌پرسد، باینری آخرین نسخه را
دانلود می‌کند و یک سرویس `systemd` می‌سازد و اجرا می‌کند. تمام.

- وضعیت: `sudo systemctl status hidybot`
- لاگ زنده: `sudo journalctl -u hidybot -f`
- حذف: `curl -fsSL https://raw.githubusercontent.com/Alighaemi9731/hiddify-free-bot/main/uninstall.sh | bash`
  (برای حذف کامل داده‌ها: `... | bash -s -- --purge`)

> آیدی عددی خودت را نمی‌دانی؟ به ربات `/myid` بفرست.

## 🛠 شروع به کار (بعد از نصب)

۱) در تلگرام به ربات `/start` بده (به‌عنوان ادمین، منوی مدیریت را می‌بینی).
۲) **🖥 مدیریت پنل‌ها → ➕ افزودن پنل**: لینک ادمین پنل را بفرست، سپس یک نمونه لینک ساب یک کاربر،
   سپس سقف حجم روزانه‌ی آن پنل (گیگ، ۰ = نامحدود).
۳) **📣 کانال‌های تبلیغ → ➕ افزودن کانال**: یک پست از کانال را forward کن (یا `@username` یا
   آیدی عددی)، سپس **تعداد جوین هدف** را بده (۰ = کانال دائمی/همیشگی).
   ⚠️ ربات باید در آن کانال **ادمین** باشد تا بتواند عضویت را بررسی کند.
۴) **⚙️ تنظیمات**: تعداد کانال نمایشی روزانه، حجم‌ها، پشتیبانی و... را تنظیم کن.

حالا کاربران می‌توانند با «🎁 دریافت کانفیگ امروز» هر روز یک کانفیگ بگیرند.

### منطق ممبرگیری
وقتی کانالی سهمیه‌ی مثلاً ۱۰۰۰ جوین دارد، ربات فقط کسانی را می‌شمارد که **قبل از نمایش، عضو
نبوده‌اند و حالا عضو شده‌اند**. به محض رسیدن به سقف، آن کانال دیگر به کاربران نشان داده نمی‌شود.
کانال‌ها بر اساس **اولویت** انتخاب می‌شوند تا سفارش‌های مهم‌تر زودتر تکمیل شوند.

## 🔌 جزئیات API هیدیفای (برای توسعه‌دهنده)

- پایه: `https://<domain>/<admin_proxy_path>/api/v2`
- احراز هویت: هدر `Hiddify-API-Key: <admin_uuid>`
- ساخت کاربر: `POST /admin/user/` با `name, usage_limit_GB, package_days, mode, telegram_id`
- حذف: `DELETE /admin/user/{uuid}/`
- مسیر ساب کاربر در API نیست؛ از نمونه لینک ساب استخراج می‌شود.

## 🧰 ساخت از سورس

```bash
git clone https://github.com/Alighaemi9731/hiddify-free-bot
cd hiddify-free-bot
go build -o hidybot ./cmd/bot

BOT_TOKEN=xxx ADMIN_ID=123456 DATA_DIR=./data ./hidybot
```

تست‌ها: `go test ./...` (تست زنده‌ی پنل با `HIDDIFY_LIVE=1` فعال می‌شود).

## 🏗 معماری

```
cmd/bot            entrypoint
internal/config    خواندن BOT_TOKEN / ADMIN_ID
internal/db        SQLite (users, panels, channels, referrals, settings) + backup/restore
internal/hiddify   کلاینت API هیدیفای v2 + parse لینک‌ها
internal/bot       منوها، فلوی دریافت کانفیگ، پنل مدیریت، FSM
internal/membership منطق شمارش جوین جدید (داخل bot)
internal/scheduler  cron: ریست روزانه تهران، بکاپ هر ۲ ساعت، پاک‌سازی
internal/backup     بکاپ فشرده + ارسال تلگرام
```
